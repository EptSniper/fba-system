# Deps for the hourly Keepa burst collector (.github/workflows/keepa-collect.yml).
# collect_hourly.py deliberately reuses the SAME scoring/gates pipeline.py uses (no parallel
# reimplementation, single source of truth) — that pulls in scikit-learn/joblib/numpy for the
# model blend, and the keepa package for the client itself. Deliberately EXCLUDES scout/
# requirements.txt's optional extras (anthropic, mcp, fastapi/uvicorn, sentence-transformers,
# pyarrow) — none of those are reachable from collect_hourly.py's import chain, and the
# cloud-inbox flush path (raw_inbox.py) zstd-compresses JSON directly, never touching pyarrow/
# Parquet (that only happens locally, in scout/drain_inbox.py, via scout/requirements.txt).
#
# ML libs PINNED to the SAME versions as requirements-train.txt (ML audit fix, 2026-07-09):
# the collector deserializes the trainer's model.joblib every run for SHADOW scoring (doctrine
# §5) — a pickled sklearn/lightgbm artifact loaded under a different library version can fail
# or silently misbehave, and version skew between the two jobs is exactly how that happens.
# Bump BOTH files together, deliberately.
keepa==1.5.0  # pinned exact — see scout/requirements.txt's comment (review fix, 2026-07-07)
