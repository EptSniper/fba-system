# Keepa Read: Amazon ~60% Buy Box + Offer Count Crash (40 → 2)

## Signal 1 — Amazon holds the Buy Box ~60% of the year

Almost always a be-very-careful flag. When Amazon is on the listing and winning the Buy Box, you compete against Amazon itself: Amazon almost always wins when in stock, won't lose price wars, and your sell-through is suppressed (you only sell in their out-of-stock windows). 60% isn't 90–100% (which is a no-go), so there's a recurring OOS gap — but it demands caution. Pull the Buy Box history graph and see how the 40% absence is distributed: one long stretch (risky) vs regular repeating dips (the profile that can work for 3P).

## Signal 2 — Offer count dropped from 40 to 2

Depends on WHY:
- **Supply dried up / sold through** (price held or rose while offers collapsed) → cautiously interesting.
- **Amazon stepped back in** (offer crash coincides with Amazon retaking the Buy Box + price drop) → warning.
- **Gating / IP enforcement** (collapse to near-zero, no price relief, never recovers) → verify eligibility first.

## Read them together

Overlay the timing of the 40→2 crash against the Buy Box and price lines:
- Crash + Amazon takes Buy Box + price drops → Amazon squeezed everyone out. Lean pass.
- Crash + price holds/rises + Amazon still OOS → real sell-through. Closer look.
- Crash + no price change + never recovers → suspect gating/IP. Verify eligibility.

## Bottom line

Amazon at ~60% Buy Box is a yellow-to-red flag; the 40→2 crash is meaningful but ambiguous. Most likely: Amazon dominates, came back in stock, and pushed 3P from 40 to 2 — I'd pass unless you have a cheaper repeatable source and only sell into documented OOS windows. Eligibility and the final buy/no-buy are separate steps; no purchase on this read alone.
